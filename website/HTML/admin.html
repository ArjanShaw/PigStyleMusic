How it works:

  • Genre (Required): Select or add a genre before scanning. Counter resets when genre changes.
  • Sublocation (Required): Choose N/A, a specific position (Left/Right Top/Bottom), or enter a custom sublocation.
  • Main Location: Choose Bin, Display, Wall, or Custom + enter identifier

How it works:

  • Last Seen Filter: Set a cutoff date - only records seen on or after that date will be posted
  • Select a Location: Choose a location from the dropdown
  • Review Records: See all records currently assigned to that location
  • Discogs Price: Calculated using the markup rules based on record age
  • Post to Discogs: Click "Post to Discogs" to list all eligible records
  • Repost Entire Store: Click the yellow button to post ALL active store records (respects last_seen filter)
  • Single Post: Use the "Post" button next to individual records to test a single listing

Formula: Discogs Price = Store Price × (1 + Markup% / 100)

💡 Tip: Only records with status "Active" and last_seen on/after cutoff date can be posted.

Custom Labels

How to Use Custom Labels

Enter one label per line. Each line will be printed on a separate label using the same dimensions as price tags. Supports multiple lines per label by using the format: Line 1|Line 2|Line 3 (use pipe | to separate lines).

Start Printing At:
Labels will start at position 1, 1 on the sheet

Enter Custom Labels

Use | (pipe) to create multiple lines on a single label. Each line will be centered separately.

Preview

0 labels

Enter labels above to see preview

Label Printing Information:

  • Uses the SAME label dimensions as price tags (configured in Admin Config)
  • Each line of text is centered on the label
  • Use | (pipe) to create multi-line labels (up to 5 lines)
  • Starting position (Row, Column) determines where on the sheet printing begins
  • Labels are arranged left-to-right, top-to-bottom on the sheet
  • Sheet layout: 4 columns × 15 rows = 60 labels per page
